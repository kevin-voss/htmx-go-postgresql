# Flow — First-time onboarding

## Diagram

```text
Login / registration
    ↓
No workspace membership found
    ↓
Create workspace
    ↓
Create first project
    ↓
Open project board
```

## Minimal required inputs

- workspace name
- workspace slug
- first project name

Keep friction low — do not ask for optional settings during onboarding.

## Outcomes

- User becomes **Owner** of the new workspace.
- At least one project exists.
- User lands on the project board / project page.
